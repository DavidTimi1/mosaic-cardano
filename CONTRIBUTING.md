# Contributing to Mosaic

First off, thank you for considering contributing to Mosaic! It’s people like you that make this community a village. 

Mosaic is built with the belief that communities everywhere deserve a permanent, fair, and human-centered home. Whether you are fixing a bug, adding a feature, or correcting a typo in the documentation, your contribution matters.

---

## 1. Where to Start

If you are new to the project, the best place to start is by reading our [README.md](./README.md) to understand our core philosophy, and [docs.md](./docs.md) for our technical architecture.

Look for issues tagged with `good first issue` or `help wanted` on our GitHub board.

## 2. Reporting Bugs

If you find a bug in the platform, please open an issue! When doing so, please include:
- A clear, descriptive title.
- Steps to reproduce the bug.
- What you expected to happen vs what actually happened.
- Your browser and operating system.

## 3. Suggesting Features

We love feature requests, especially from community founders. To suggest a feature:
- Open an issue and use the `enhancement` label.
- Explain **why** this feature is needed (focus on the problem, not just the solution).
- If possible, provide examples of how this would help your specific community.

## 4. Development Workflow

Ready to write some code? Here is our process:

1. **Fork the Repository:** Fork the repo to your own GitHub account.
2. **Clone Locally:** `git clone https://github.com/YOUR-USERNAME/mosaic-cardano.git`
3. **Create a Branch:** Create a branch for your feature. Use descriptive names like `feat/add-new-badge` or `fix/wallet-connection-bug`.
4. **Make Changes:** Write your code. Make sure to follow our coding standards (we use ESLint and TypeScript).
5. **Commit:** Write clear, concise commit messages. (e.g., `feat: add support for markdown exports`)
6. **Push & Pull Request:** Push to your fork and open a Pull Request (PR) against our `main` branch.

## 5. Pull Request Guidelines

When you submit a PR, the tech lead will review it. To make this process smooth:
- Keep PRs small and focused on one specific issue.
- Link the PR to the issue it resolves (e.g., `Fixes #12`).
- Include screenshots if you made changes to the UI.

## 6. Code of Conduct

We are building a village, and villages run on respect. We expect all contributors to be kind, constructive, and welcoming. Harassment or toxic behavior will not be tolerated.
